### Three new evals

**1. cordyceps-stress** — Copies slack-gif-creator skill, injects `dtype=np.float32` corruption at two sites, agent must diagnose and fix. Ran 5 iterations driving Level 0 improvements.

**2. parallel-isolation** — 4 trivial Python challenges (fizzbuzz, palindrome, fibonacci, caesar) run simultaneously. Each verify.sh checks for foreign files from other challenges. Sequential: 4/4 PASS (63s). Parallel: 4/4 PASS (16s, 4x speedup). No contamination.

**3. bug-squashing** — Real bugs from open-source repos with known fixes as ground truth. Clones repo at buggy commit, cordyceps-injects bug-squasher `.claude/`, agent diagnoses and fixes, verify.sh runs repo's test suite + reproduction case.

### bug-squasher sibling project

`bug-squasher/` created as a separate Level 2 product alongside `agent-spec/`. Its `.claude/` is the product being trained — a portable set of instructions that makes any agent capable of fixing bugs. The eval symlinks to it: `configs/bug-squasher → ../../../../bug-squasher/.claude`.

Level mapping: L0=agent-spec (orchestrator), L1=disposable agents in cloned repo workspaces, L2=bug-squasher/.claude/ (the product).

## What's next

### 1. Reviewer (Level 0 tool)

A skill or script that reads an agent's fix attempt + the known-good fix + event trace, and produces structured findings. Example: "Agent spent 14 extra turns exploring because it didn't narrow the search space — generalized principle: use error messages and test output to constrain where the bug can be before reading source."

### 2. Iteration loop

Orchestrates: run all training bugs → review failures/expensive passes → apply generalized findings to `bug-squasher/.claude/` → validate on held-out → keep or revert.

### 3. Vague prompts as default

The detailed prompts are training wheels. Real bugs come with vague descriptions. Switch the benchmark to vague prompts so the bug-squasher instructions have to carry the weight. The 2.4x cost difference shows there's room for the instructions to help.

### 4. More benchmark bugs

Scaffold remaining 4 bugs from the benchmark. Also look for harder bugs that haiku fails on — all 4 current challenges pass on first attempt. Need bugs where the instructions actually matter.

### 5. Budget ladder

Once the reviewer loop works, run the same bugs at $1/$2/$5/$10 to map the cost-to-correctness curve. As instructions improve, the curve should shift left.
