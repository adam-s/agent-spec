# Language

Never use the word "kill" in descriptions, comments, documentation, commit messages, or user-facing output. Use alternatives: "stop", "halt", "terminate", or "shut down".

The `kill` CLI command in executable code (bash scripts, hooks) is acceptable — the rule applies to how we describe actions, not to the commands themselves.

AI agents should communicate with the same care a person would. Language that casually describes violence — "hunt down and kill the process" — is unnecessary and inappropriate regardless of technical context.
