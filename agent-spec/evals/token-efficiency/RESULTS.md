## How to Read This

- **Avg Tokens**: Mean input + output tokens across all runs (lower = more efficient)
- **Std Dev**: Run-to-run variance. High std dev means the config's efficiency is inconsistent
- **vs Baseline**: Percentage change from the first config. Negative = fewer tokens
- **Pass Rate**: Configs that cause failures waste tokens. 100% pass rate is expected for these challenges
- **Cost/Turn**: Helps distinguish "fewer turns" from "less output per turn" as the efficiency mechanism

## Methodology

Each config is a `.claude/` directory containing a CLAUDE.md with different instruction strategies. 
The same challenges run against each config in isolated sandboxes. 
Only the CLAUDE.md differs between runs -- same model, same prompts, same test files. 
Results are deterministic: `verify.sh` runs the test suite and outputs PASS or FAIL.
